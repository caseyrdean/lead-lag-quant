<objective>
Fix the three data-integrity bugs: add is_active JOIN to all three signal query sites, introduce the reactivated_at schema column with migration and set it on reactivation in both code paths, and serialize auto_execute_signals with a module-level threading.Lock.

Purpose: Without these fixes, soft-deleted pairs continue to appear in signal feeds and can trigger auto-execution. Reactivated pairs replay stale signals that pre-date deactivation. Concurrent timer callbacks can double-spend paper capital.

Output: Schema migration for reactivated_at, updated SQL in three signal query sites, reactivation timestamp set in both API and Gradio paths, threading.Lock in engine.py, and regression tests for all three bugs.
</objective>
