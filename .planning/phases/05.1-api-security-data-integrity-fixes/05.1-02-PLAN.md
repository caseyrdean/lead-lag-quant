<objective>
Fix the three data-integrity bugs in the FastAPI layer and shared data layer only. The Gradio UI (ui/) is being deleted in Plan 05.1-04 and must NOT be modified here.

Fixes:
1. Add is_active JOIN to the two FastAPI/shared signal query sites (api/routes/signals.py, paper_trading/db.py)
2. Introduce the reactivated_at schema column with migration and set it on reactivation in the API path only
3. Serialize auto_execute_signals with a module-level threading.Lock

Purpose: Without these fixes, soft-deleted pairs continue to appear in signal feeds and can trigger auto-execution. Reactivated pairs replay stale signals that pre-date deactivation. Concurrent API/timer callbacks can double-spend paper capital.

Output: Schema migration for reactivated_at, updated SQL in two FastAPI/shared signal query sites, reactivation timestamp set in the API path, threading.Lock in engine.py, and regression tests for all three bugs.
</objective>
