# Phase 05.1 Plan 01: API Security and Data Integrity Fixes Summary

**One-liner:** FastAPI FREE-tier pair cap (HTTPException 403), Pydantic Field(gt=0) on buy/sell shares/price, and try/except JSONResponse(500) on all analytics endpoints — with 11 regression tests via dependency_overrides TestClient fixture.

## What Was Built

Four API-layer bugs fixed across three route files; TestClient test infrastructure established with a dependency_overrides-based fixture that isolates tests from the real lifespan-managed DB, scheduler, and Polygon client.

### BUGFIX-01: FREE-tier pair limit enforced server-side

`api/routes/pairs.py` — Added `FREE_TIER_MAX_PAIRS = 5` module constant and a guard at the top of `add_pairs()` that counts active pairs and raises `HTTPException(403)` before any INSERT when `plan_tier == PlanTier.FREE`. Previously a FREE-tier user could bypass the 5-pair cap visible in the Gradio UI by calling the API directly.

### BUGFIX-02: Soft-delete commit verified present

`api/routes/pairs.py` — `delete_pairs()` already had `conn.commit()` after the UPDATE loop. Verified and added comment `# BUGFIX-02: commit is present (verified 2026-03-21)`. No behavioral change needed.

### BUGFIX-06: Pydantic input validation on buy/sell

`api/routes/trading.py` — Added `Field(gt=0)` to `BuyRequest.shares`, `BuyRequest.price`, `SellRequest.shares`, and `SellRequest.price`. FastAPI now automatically returns HTTP 422 with field-level error detail for invalid inputs (shares=0, price=-1.0) before any handler code runs. `price: float | None = Field(default=None, gt=0)` preserves the None=fetch-from-Polygon path.

### BUGFIX-07: Structured error responses on analytics endpoints

`api/routes/analytics.py` — Wrapped all six endpoint bodies (`/stats`, `/risk`, `/equity`, `/ticker-breakdown`, `/pnl-distribution`, `/monthly-heatmap`) in `try/except Exception as exc` blocks returning `JSONResponse(status_code=500, content={"error": str(exc)})`. Previously, any exception would propagate as an unhandled 500 with a raw Python traceback visible in the response.

### Test Infrastructure

`tests/conftest.py` — Added `api_client` fixture using `app.dependency_overrides` to replace `get_conn`, `get_config`, and `get_client` with test doubles. This approach is correct because `TestClient` with a lifespan-equipped app runs the lifespan on context manager entry, which would overwrite `app.state` set before entering the `with` block.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] TestClient fixture: app.state mutation overwritten by lifespan**

- **Found during:** Task 1 verification (test_free_tier_limit got 200 instead of 403)
- **Issue:** The plan specified setting `app.state.conn = tmp_db` before yielding TestClient. However, `TestClient(app)` as a context manager triggers the FastAPI lifespan, which runs and overwrites `app.state.conn`, `app.state.config`, and `app.state.client` with the real production values.
- **Fix:** Replaced `app.state` mutation with `app.dependency_overrides` targeting the three dependency callables (`get_conn`, `get_config`, `get_client`). This approach bypasses the lifespan-injected values at the dependency resolution layer.
- **Files modified:** tests/conftest.py
- **Commit:** 8bc9e4f

**2. [Rule 1 - Bug] TestClient.delete() has no body parameter**

- **Found during:** Task 2 (`test_delete_commits`)
- **Issue:** starlette 0.52.1 `TestClient.delete()` signature only accepts `params`, `headers`, `cookies` — no `json=` or `content=` body parameter.
- **Fix:** Used `api_client.request("DELETE", "/api/pairs", json={"ids": [pair_id]})` which accepts the full httpx request signature including body.
- **Files modified:** tests/test_api_pairs.py
- **Commit:** 8bc9e4f
