## Decisions Made

- Used INNER JOIN rather than LEFT JOIN + WHERE for the is_active filter — cleaner exclusion semantics at the SQL level
- reactivated_at guard uses `>=` so NULL (never deactivated) means all signals are valid; non-NULL gates on timestamp
- threading.Lock wraps the entire function body with blocking=True so the second caller waits — a non-blocking skip would silently drop execution
- Gradio UI files (ui/app.py, ui/signal_dashboard.py) intentionally not touched — being deleted in Plan 05.1-04

- **Issue:** Stale and fresh signals both had the same signal_date (today), triggering sqlite3.IntegrityError on the second INSERT due to UNIQUE(ticker_a, ticker_b, signal_date) constraint in the signals table
- **Fix:** Used dates 2 days apart (stale_date = now - 2 days, fresh_date = now + 1 day) with explicit signal_date parameter to _insert_signal()
