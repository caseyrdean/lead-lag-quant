<objective>
Fix broadcast_sync() in api/ws.py to use asyncio.run_coroutine_threadsafe instead of loop.create_task, eliminating silent message loss when WebSocket broadcasts are triggered from background threads.

Purpose: loop.create_task() is not thread-safe. When BackgroundPricePoller or PipelineScheduler calls broadcast_sync() from a worker thread, messages can be silently discarded. asyncio.run_coroutine_threadsafe() is the stdlib-correct cross-thread coroutine scheduling API.

Output: One-line fix in api/ws.py and a unit test confirming the correct API is used.
</objective>

The full corrected method:

```python
def broadcast_sync(self, event_type: str, data: dict) -> None:
    """Fire-and-forget broadcast from a sync (threaded) context."""
    try:
        loop = asyncio.get_running_loop()
        asyncio.run_coroutine_threadsafe(self.broadcast(event_type, data), loop)
    except RuntimeError:
        pass
```

`asyncio` is already imported at the top of the file (it's used for `get_running_loop`). No new imports needed.

Do NOT add a guard for "called from event loop thread" — run_coroutine_threadsafe works from both contexts. Do NOT change the RuntimeError handler.

The return value of run_coroutine_threadsafe (a concurrent.futures.Future) is intentionally discarded — this is fire-and-forget broadcast behavior.

**Test `test_broadcast_sync_uses_run_coroutine_threadsafe`:**

Strategy: mock `asyncio.get_running_loop` to return a mock loop object, then call `broadcast_sync()` and assert `asyncio.run_coroutine_threadsafe` was called (not `loop.create_task`).

```python
import asyncio
from unittest.mock import MagicMock, patch
import pytest

from api.ws import ConnectionManager


def test_broadcast_sync_uses_run_coroutine_threadsafe():
    manager = ConnectionManager()
    mock_loop = MagicMock()

    with patch("api.ws.asyncio.get_running_loop", return_value=mock_loop), \
         patch("api.ws.asyncio.run_coroutine_threadsafe") as mock_rct:
        manager.broadcast_sync("price_update", {"ticker": "AAPL", "price": 150.0})

    mock_rct.assert_called_once()
    # Verify create_task was NOT used as the scheduling mechanism
    mock_loop.create_task.assert_not_called()
```
