**BUGFIX-06: Add Field(gt=0) to BuyRequest and SellRequest in api/routes/trading.py.**

Find the `BuyRequest` and `SellRequest` Pydantic models. Add `Field(gt=0)` constraints:

```python
from pydantic import BaseModel, Field

class BuyRequest(BaseModel):
    ticker: str
    shares: int = Field(gt=0)
    price: float | None = Field(default=None, gt=0)
    source_signal_id: int | None = None
    invalidation_threshold: float | None = None

class SellRequest(BaseModel):
    ticker: str
    shares: int = Field(gt=0)
    price: float | None = Field(default=None, gt=0)
```

Preserve all existing fields — only add `Field(gt=0)` to `shares` and `price`. Do not change the endpoint logic.

Key detail: `price: float | None = Field(default=None, gt=0)` — Pydantic v2 skips `gt=0` when value is None, so `price=None` (fetch from Polygon) still works. Do NOT use `price: float = Field(gt=0)` which would reject None.
