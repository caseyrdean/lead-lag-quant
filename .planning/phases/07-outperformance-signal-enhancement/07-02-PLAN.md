**compute_rs_slope(conn, ticker_a, ticker_b, lookback_sessions=5) -> float | None**
- Queries `features_relative_strength` for the pair ordered by `trading_day DESC LIMIT lookback_sessions * 3` (extra rows for stable std dev)
- Reverses to chronological order, takes last `lookback_sessions` values as `recent`
- Computes slope via `np.polyfit(np.arange(len(recent)), recent, 1)[0]`
- Computes `rs_std` from the full fetched series (not just the recent window) — use `pd.Series.std()` on the full fetched `rs_value` column; require >= 10 rows for a valid std dev
- If `rs_std` is available and > 0: return `slope / rs_std` (normalized); otherwise return `float(slope)` un-normalized
- Return `None` if fewer than `lookback_sessions` rows available

**compute_leader_baseline_return(conn, ticker_a, optimal_lag, lookback_days=120) -> float | None**
- Queries `features_lagged_returns` for `ticker=ticker_a, lag=optimal_lag` over the last `lookback_days` from the anchor (MAX trading_day for that ticker)
- Returns `float(df['return_value'].mean())` or `None` if no rows
- Use: `date(anchor, '-120 days')` as the WHERE lower bound

**compute_response_window(conn, ticker_a, ticker_b) -> float | None**
- Queries all rows from `signal_transitions` for the pair ordered by `transitioned_at ASC`
- Returns `None` if no rows (bootstrap — no history yet)
- Identifies BUY entry events: `to_action = 'BUY'` and (prior row has `to_action != 'BUY'` OR it is the first row)
- For each BUY entry, find the next row where `to_action != 'BUY'` (the BUY exit)
- For each complete BUY→exit pair, count trading sessions using:
  `SELECT COUNT(DISTINCT trading_day) FROM normalized_bars WHERE ticker=? AND trading_day > ? AND trading_day <= ?`
  where ticker is `ticker_b` (follower), the dates come from `transitioned_at` values (strip time component, use date portion only)
- Compute mean of all measured BUY durations
- Return `None` if fewer than 2 complete BUY→exit cycles exist; otherwise return the float mean

Logic (in order, return on first match):
1. If `len(rs_series) < n_sessions`: return `'HOLD'`
2. `recent = rs_series.iloc[-n_sessions:]`
3. BUY condition 1 — consistent outperformance: `if (recent > 0).all(): return 'BUY'`
4. BUY condition 2 — reversal: check `len(rs_series) >= n_sessions + 1`. Get `pre_reversal = rs_series.iloc[-(n_sessions + 1)]`. If `pre_reversal < 0` AND `(recent > 0).all()`: return `'BUY'`
   - CRITICAL: BUY condition 1 is checked before condition 2, so condition 2 only fires when recent is already positive — the reversal check just adds the prior-negative gate. Do not duplicate the `(recent > 0).all()` check.
5. SELL condition: `diffs = recent.diff().dropna()`. If `len(diffs) == n_sessions - 1` AND `(diffs < 0).all()`: return `'SELL'`
6. HOLD: `current_rs = rs_series.iloc[-1]`. If `abs(current_rs - rs_mean) <= rs_std`: return `'HOLD'`
7. Default: return `'HOLD'`
