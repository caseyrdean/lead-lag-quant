**frontend/src/components/backtest/XcorrHeatmap.tsx**
Props: data (XcorrHeatmapPoint[]).
Use HTML table technique from MonthlyHeatmap.tsx — NOT a Recharts component (too many SVG elements for 11 × 60 cells).
Rows = lags -5 to +5 (11 rows), columns = unique trading_days from data (most recent 30 shown by default to keep table width manageable — show last 30 unique dates in the data).
Cell background color via corrColor helper: positive correlation = rgba(34, 197, 94, ...) green, negative = rgba(239, 68, 68, ...) red, intensity = Math.min(Math.abs(value), 1). Null = transparent.
Significant cells (is_significant=1) get a subtle border or dot indicator.
If data empty, show "No cross-correlation data — run the pipeline first".
Row header = lag label (e.g., "Lag -5", "Lag 0", "Lag +5"). Column headers = date strings (abbreviated: "MM/DD").

**frontend/src/components/backtest/RollingOptimalChart.tsx**
Props: data (XcorrHeatmapPoint[]).
From the full XcorrHeatmapPoint[], derive the "optimal correlation" per trading_day: for each date, find the lag with maximum Math.abs(correlation) among significant rows (is_significant=1). If none significant, use max absolute across all lags.
Render as a Recharts LineChart with ResponsiveContainer. X-axis = trading_day, Y-axis = optimal correlation value (-1 to 1). Line in accent color. Tooltip showing lag and correlation on hover.
If no data, show "No rolling correlation data" placeholder.
Limit to last 90 unique trading_days from data to avoid dense rendering.
