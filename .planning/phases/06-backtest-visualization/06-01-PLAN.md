Create `backtest/__init__.py` as an empty file (package marker only).

Create `backtest/engine.py` with three pure functions. Never import from ingestion_massive/ or call PolygonClient anywhere in this file.

**run_backtest(conn, leader, follower, start_date, end_date) -> dict**
- BACKTEST-01: reads only from SQLite.
- BACKTEST-02: signal selection is the primary look-ahead bias control — only include signals WHERE ticker_a=leader AND ticker_b=follower AND signal_date BETWEEN start_date AND end_date. For any scenario requiring split re-computation, filter splits WHERE fetched_at <= signal_date.
- BACKTEST-03: compute and return all four metrics.
- Return dict keys: leader, follower, start_date, end_date, total_trades, winning_trades, hit_rate, mean_return_per_trade, annualized_sharpe, max_drawdown.
- Return zero-dict (all numeric fields = 0.0, total_trades=0) if no signals found in range.
- Use features_lagged_returns to resolve return at the signal's optimal_lag offset (avoids calendar vs. trading day arithmetic pitfall from research). Query: SELECT return_value FROM features_lagged_returns WHERE ticker=ticker_b AND trading_day=signal_date AND lag=optimal_lag. If row missing, skip that signal.
- Metric computation: mirror paper_trading/analytics.py exactly.
  - hit_rate: count(ret > 0) / total_trades (as decimal 0-1, not percentage)
  - mean_return_per_trade: sum(returns) / len(returns)
  - annualized_sharpe: (mean / std) * math.sqrt(252) if std != 0 else 0.0
  - max_drawdown: cumsum then cummax pattern from analytics.py; return as negative decimal (e.g., -0.15)
- Use get_logger("backtest.engine") from utils.logging.

**xcorr_data(conn, leader, follower, days=60) -> list[dict]**
- Query features_cross_correlation WHERE ticker_a=leader AND ticker_b=follower AND trading_day >= date('now', f'-{days} days') AND correlation IS NOT NULL ORDER BY trading_day ASC, lag ASC.
- Return list of dicts: {lag, trading_day, correlation, is_significant}.
- Return empty list if no data (handles fresh DB gracefully).

**regime_state(conn, follower) -> dict**
- Query regime_states JOIN distribution_events for most recent row for the follower ticker (see SQL in research Pattern: Regime State Endpoint Data Shape).
- If no rows found, return sentinel: {"regime": "Unknown", "trading_day": None, "rs_value": None, "price_vs_21ma": None, "price_vs_50ma": None, "atr_ratio": None, "volume_ratio": None, "vwap_rejection_streak": None, "is_flagged": 0}.
