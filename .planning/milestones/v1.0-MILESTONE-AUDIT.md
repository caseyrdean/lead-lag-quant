## Critical Gap: ENGINE-03

**Requirement:** Hard threshold gate — signals generated ONLY when `stability_score > 70 AND correlation_strength > 0.65`

**What was found:**

`signals/generator.py` contains:
```python
STABILITY_THRESHOLD = 50.0
CORRELATION_THRESHOLD = 0.50
```

The gate logic uses strict `>` inequalities (correct), but the threshold values are `50.0` / `0.50` — significantly below the specified `70` / `0.65`. This means the signal gate is more permissive than the milestone requires: signals with stability scores between 50–70 or correlations between 0.50–0.65 would be generated, despite the requirement prohibiting them.

**Corroborating evidence:**
- `tests/test_signals_generator.py` has 12 pre-existing failures — consistent with tests written for the 70/0.65 spec running against code with 50/0.50 thresholds
- Phase 4 VERIFICATION marked ENGINE-03 SATISFIED (verified Feb 2026) but the integration checker read the current live file and found the lower values
- The Phase 4 verifier noted "uses abs(correlation_strength) instead of raw value" as an INFO deviation but did not flag the threshold values specifically

**Impact:** Signal quality guarantee in the milestone definition is not met. The hard gate is the primary quality control mechanism.
